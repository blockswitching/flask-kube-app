About Docker

Understanding containers and why they matter

What is Docker?

Docker is a platform that lets you package your application and all its dependencies into a standardized unit called a container. Think of it like a shipping container for your code — it works the same everywhere, regardless of the host system.

Key Concepts

Image

A read-only template with instructions for creating a container. Like a recipe for your app's environment.

Container

A running instance of an image. It's an isolated process with its own filesystem, networking, and resources.

Dockerfile

A text file with instructions that Docker uses to build an image. Each instruction creates a layer.

Registry

A storage location for Docker images. Docker Hub is the default public registry.

Our Dockerfile Explained

# Start from Python 3.9 slim base image
FROM python:3.9-slim

# Set the working directory inside the container
WORKDIR /app

# Copy requirements first (for better layer caching)
COPY requirements.txt .

# Install Python dependencies
RUN pip install --no-cache-dir -r requirements.txt

# Copy the rest of the application code
COPY . .

# Tell Docker this container listens on port 5001
EXPOSE 5001

# Command to run when the container starts
CMD ["gunicorn", "--bind", "0.0.0.0:5001", "app:app"]

Common Docker Commands

Command Description
docker build -t name . Build an image from a Dockerfile
docker run -p 5001:5001 name Run a container from an image
docker ps List running containers
docker images List available images
docker stop <id> Stop a running container
